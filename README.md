# leiting-hiring

BOSS 直聘招聘助手，基于 web-access CDP 接管浏览器，支持简历筛选、站内沟通、飞书约面三类服务。

## 前置依赖

使用前需要完成以下三项安装，每台电脑只需装一次。

### 1. Node.js 22+

下载地址：https://nodejs.org/

安装完成后验证：
```bash
node -v  # 应显示 v22.x.x 或更高
```

### 2. web-access skill

```bash
git clone https://github.com/eze-is/web-access ~/.claude/skills/web-access
```

### 3. Chrome 开启远程调试

在 Chrome 地址栏打开 `chrome://inspect/#remote-debugging`，勾选 **Allow remote debugging for this browser instance**。

> 每次重启 Chrome 后需要重新勾选。

1. 完成上方三项前置依赖
2. 打开并登录 BOSS 直聘招聘者页面（`https://www.zhipin.com/web/chat/index`）

## 核心设计原则

- **启动轻**：静默探针，环境 OK 直接进
- **防封号**：消息只走页面按钮、安全验证立即停、每日发消息上限 30 条
- **人工确认**：筛选结果给用户看，用户确认后才发消息
- **持久化**：候选人记录跨会话保留，不重复处理
- **站点经验**：BOSS DOM 路径已验证，不重复探索

## 触发词

- 「帮我筛简历」「筛一下投递」→ 进入 screening
- 「帮我联系这些候选人」「发消息」→ 进入 chat
- 「帮我约面」→ 进入 scheduling

## 更新日志

### v1.1.0（2026-04-24）

本次更新来自实际操作中发现的问题，全部针对真实踩坑场景。

**再也不会发出乱码消息**
通过 CDP 向输入框写中文会产生乱码，乱码消息会触发 BOSS 风控。现在所有消息只走页面上的按钮（求简历按钮、默认打招呼按钮），彻底杜绝乱码。

**遇到安全验证立即停止，不再继续操作**
之前安全验证弹出后操作仍在继续，存在封号风险。现在只要出现验证码或安全验证弹窗，立即停止全部操作并通知用户，等用户处理完再恢复。

**不会重复给同一个人发求简历**
发送「求简历」前会先检查聊天记录，如果已经发过、或候选人已经发来附件，直接跳过，不重复打扰。

**专升本候选人会被正确识别为大专学历**
之前只看学历标签，「专升本」字样藏在教育经历文字里会被漏掉，导致误判为本科通过筛选。现在读取完整教育经历文字，专升本正确归类为大专。

**期望岗位不再影响筛选结果**
候选人填的期望岗位与实际投递岗位不一致很常见，不应作为筛选依据。现在忽略期望岗位字段，只看实际经历和技能。

**启动时一次性授权，不再逐条弹提示**
每次操作都弹授权确认很打断节奏。现在启动时统一授权一次，本次会话内所有操作不再重复询问。

**筛选速度提升，明显不合适的快速跳过**
之前每个候选人都有固定停顿，拖慢整体速度。现在明显不符合要求的直接跳过，有潜力的看 3-5 秒，节奏更接近人工操作。

**过程中不再中途暂停等待确认**
之前每发送 3 人就暂停等用户确认，打断操作流。现在全程连续执行，只在筛选汇报后才等用户决策。
